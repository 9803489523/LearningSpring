    Жизненный цикл бина:
запуск спринг-приложения,
инициализация объекта бина,
вызов init-метода,
использование бина,
вызов destroy-метода
остановка спринг-приложения

    init - запускается в ходе инициализации бина,
для singleton запускается один раз,
для prototype - столько раз, сколько будет вызван метод getBean()
    destroy - в ходе уничтожения,
для singleton запускается один раз в конце
для prototype не предусмотрен

    как init, так и destroy мы прописываем в классе бина
к этому методу лишь одно требование: он  не должен содержать входных параметров

    factory-method - метод, предлагающий создавать объекты
не напрямую через new, а через вызов особого фабричного метода
Объекты все равно будут созданы с помощью new, но делать
это будет фабричный метод (иногда бывает полезно, например
в тех случаях, когда в классе приватный конструктор), он
в классе бина должен быть статическим.
в Singleton фабричный метод вызовется один раз!
все вызовы getBean
